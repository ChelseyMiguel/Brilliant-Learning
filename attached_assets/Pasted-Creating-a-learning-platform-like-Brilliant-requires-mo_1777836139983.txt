Creating a learning platform like **Brilliant** requires more than just a standard "course" layout. Brilliant’s magic lies in its **active learning** philosophy—replacing long lectures with immediate, bite-sized challenges and visual feedback.

Here is a comprehensive prompt you can use with an AI (like myself or a coding assistant) to architect the foundation of a Brilliant-style learning application.

---

## The Master Prompt

> **Role:** You are a Senior Full-Stack Developer and Educational UX Designer specializing in "Active Learning" platforms.
>
> **Task:** Design the technical architecture and a functional prototype for a web-based learning platform inspired by Brilliant.org.
>
> **Core Principles to Implement:**
> 1. **Interactive First:** No long videos or walls of text. Concepts must be taught through immediate problem-solving.
> 2. **Scaffolding:** Each lesson starts with a simple "low floor" intuition check and builds to a "high ceiling" complex problem.
> 3. **Instant Feedback:** Users should receive immediate visual or textual feedback upon submitting an answer.
> 4. **Gamification:** Include a progress map, daily streaks, and "XP" logic.
>
> **Technical Requirements:**
> * **Frontend:** React or Next.js with Tailwind CSS for a clean, minimalist aesthetic.
> * **State Management:** Use a library (like Zustand or Redux) to track user progress through a "Learning Path."
> * **Components Needed:** >     * A **Progress Tracker** (visual node-based map).
>     * An **Interactive Card** component that supports multiple-choice, drag-and-drop, or numeric input.
>     * An **Explanation Toggle** that reveals the "Why" only after an attempt is made.
>     * **LaTeX Integration** for rendering beautiful mathematical formulas (e.g., using KaTeX).
>
> **Specific Content Example:**
> Please generate the first lesson of a "Probability & Logic" course. Create three interactive cards that teach the concept of the "Gambler's Fallacy" through a coin-toss simulation.
>
> **Output:** Provide the system architecture diagram (text-based), the database schema for courses/users, and the React code for the main interactive lesson interface.

## Key Features to Focus On

If you are building this, prioritize these three elements to capture that "Brilliant" feel:

### 1. The "Interactive Story" UI
Instead of a sidebar with 50 links, use a **Linear Path**. This reduces "choice paralysis." Users should see a trail of nodes (lessons) that they unlock one by one.

### 2. High-Fidelity Feedback
Brilliant doesn't just say "Incorrect." It often shows a visual representation of *why* the answer was wrong.
* **Math:** If they miss a geometry question, show the shape deforming.
* **Code:** If the logic is wrong, show the data flow breaking.

### 3. The "Intuition" Hook
Every lesson should start with a question that seems easy but challenges a common misconception. For example:
> *“If you flip a fair coin 5 times and get Heads every time, what is the probability the next flip is Tails?”*
