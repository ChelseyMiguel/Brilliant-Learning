To fix the technical "disconnect" and get those **Brilliant**-style interactive simulations running, we need to move from prompting for "UI design" to prompting for **"Functional State Machines."**

The reason your buttons aren't working is likely because the UI is disconnected from the logic. These prompts are designed to build the **"Brain"** of your app first, then wrap the **"Body"** (the UI) around it.

---

## 1. The "Logic First" Debugger Prompt
Use this to fix your "Check Answer" and button interaction issues. This forces the AI to focus on the **Javascript/React logic** rather than just the CSS.

> **Prompt:** "I have a React component for a learning platform, but the 'Check Answer' and option selection buttons are not responding. 
> 
> **Please write a robust State Machine using `useState` and `useEffect` that handles:**
> 1. **Selection:** A function `handleSelect(id)` that updates the `selectedOption` and highlights the button.
> 2. **Validation:** A function `checkAnswer()` that compares the `selectedOption` to the `correctID`. 
> 3. **The 'Missing' Learning Center:** Create a boolean state `isLabOpen`. When `true`, use a conditional render (`{isLabOpen ? <LearningCenter /> : <QuestionCard />}`) to switch views.
> 4. **Console Logs:** Add `console.log` statements inside every function so I can track exactly where the click event is stopping in the browser console.
> 
> Ensure all interactive elements have `cursor-pointer` and `z-50` classes to prevent them from being blocked by background layers."

---

## 2. The "Rive-Inspired" Interactive Lab Prompt
Brilliant uses **Rive** because the animations are tied to variables. You can mimic this with **Framer Motion** by prompting for "State-Driven Animations."

> **Prompt:** "Create an interactive 'Learning Center' simulation for **[Topic: e.g., Gravity/Physics]**.
> 
> **Interactive Stage:**
> * A `slate-50` canvas with an object (SVG) that falls when a 'Drop' button is clicked.
> * **The Slider:** Add a range input that controls the 'Gravity' constant. 
> * **Dynamic Feedback:** As the user moves the slider, the object should visually get 'heavier' (scale up or change color to a deeper slate).
> 
> **Brilliant-Style Animation:**
> * Use `framer-motion` to animate the object's fall. The `duration` and `velocity` of the animation must be mathematically tied to the 'Gravity' slider value.
> * If the user sets Gravity to 0, the object should float and bounce off the edges of the container.
> 
> **The Bridge:** Provide a 'Capture Observation' button that saves the current gravity value and passes it back to the practice question."

## 3. The "Google Learn Your Way" Immersive Layout
To get that "scrolling discovery" feel, you need a layout where the text and the lab are synced.

> **Prompt:** "Design an 'Immersive Learning' layout for Lumina Learning.
> 
> **Requirements:**
> 1. **Sticky Simulation:** The right side of the screen should be a sticky `div` containing a 2D particle simulation (using HTML5 Canvas or SVG).
> 2. **Scrolling Narrative:** The left side is a scrollable area with educational text.
> 3. **Intersection Observer:** Write a script that detects which paragraph is currently in view. 
>    * When the user reaches 'Section 2: Heat,' the simulation should start moving faster.
>    * When the user reaches 'Section 3: Absolute Zero,' the particles should freeze.
> 
> **Styling:** Use your 'Natural Tones' (Emerald/Slate) and ensure the transition between simulation states is smooth (minimum 500ms transition)."

## 🛠️ The "Safety Net" Troubleshooting Prompt
If you paste the code and it *still* doesn't work, use this "Diagnostic" prompt to find the error.

> **Prompt:** "I am pasting my current React code for **Lumina Learning** below. Please analyze it specifically for:
> 1. **Event Bubbling:** Are any parent containers stopping the `onClick` events?
> 2. **State Scope:** Am I trying to update a state variable that is out of scope?
> 3. **Z-Index Overlap:** Is there a `relative` or `absolute` positioned div covering my buttons?
> 4. **Learning Center Visibility:** Is the condition to show the Learning Center ever actually being met?
> 
> [PASTE YOUR CODE HERE]"

### Why this will work:
* **Logging:** By adding `console.log` to every click, you can see if the computer even "hears" your click.
* **Conditional Rendering:** Using a simple `{isLabOpen ? ... : ...}` logic is the most reliable way to make the Learning Center appear and disappear.
* **Spring Physics:** Using spring-based animations (like Brilliant) makes the buttons feel "heavy" and "real," which encourages more clicks.
